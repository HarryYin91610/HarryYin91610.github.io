个人网站部署笔记

发表于 2019-02-19 | 分类于 Deploy

生产环境要素

  1. 购买自己的域名
  2. 购买自己的服务器
  3. 域名备案（在国内的网站都必须备案）
  4. 配置服务器应用环境
  5. 安装配置数据库
  6. 项目的部署与发布

搭建流程

1. 购买域名

  • 厂商选择：阿里云（万网）、爱名网、dnspod，godaddy。为方便备案最终选择阿里云的域名；
  • 搜索想要的域名前缀，选择纯英文的域名；
  • 后缀优先选择： .cn， .com， .cc；
  • 域名需要实名认证。

2. 购买服务器

  • 厂商选择：亚马逊AWS，阿里云ECS，Linode，Heroku，DigitOcean，青云，UCloud等；
  • 同样为方便备案选择国内的阿里云；
  • 公共镜像：Ubuntu 14.04 64位；
  • 设置root密码

3. 域名备案

  • 在阿里云直接进行备案，备案完成前首次备案的域名在线上是不允许访问的；
  • 首先申请备案服务号；
  • 完成备案流程

4. ssh远程登录服务器

4.1 root登录

  • 在mac终端，使用root身份登录：

    a. 输入：ssh root@服务器公网ip；
    b. 输入之前在服务器设置的密码；
    c. 出现欢迎信息即完成登录。

  • 查看硬盘使用的命令

  • 退出登录

4.3 ssh配置无密码登录

4.3.1 在本地终端

  • 进入.ssh文件夹

  • 查看是否有id_rsa和id_rsa.pub

  • 若没有，则继续下面的命令

  • 连续回车，生成公钥和私钥

  • 进入.ssh，打印公钥内容

  • 运行ssh代理

  • 将本机的私钥加入到代理中

4.3.2 在服务器端
  • 登录服务器

  • 同样生成公私钥

  • 同样运行ssh代理

  • 同样将服务器上的私钥加入到代理中

  • 进入.ssh，生成授权文件

  • 将刚刚本地生成的公钥写入authorized_keys文件

  • 给authorized_keys授权

  • 重启ssh服务

  • 完成用户免密登录

4.4 修改服务器默认登录端口

  • 登录服务器

  • 修改配置文件

  • 在sshd_config文件里，修改端口（默认22，不要使用0~1024， 可使用：1025~65535）

  • 在sshd_config文件里，末尾增加一行（用户名是服务器刚刚添加的用户）

  • 保存，退出sshd_config文件

  • 重启ssh

  • 进入sshd_config

  • 关闭root密码登录

  • 关闭密码授权

  • 重启ssh
